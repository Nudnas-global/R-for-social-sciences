Social Science Data Analysis in R – Training Package

Prepared by: Dr Sandun Dassanayake

Training Overview

This 3-hour training is designed for academic staff in the social sciences to get familiar with the data analysis techniques using R and RStudio. It uses example datasets of student evaluations and survey responses (with Likert-scale and categorical variables) to practice core data analysis skills. The training is organized into five modules, each with an instructor guide and accompanying presentation slides:

  • Module 1: Introduction to R and RStudio (basic syntax and using packages)
  • Module 2: Data import (CSV/Excel), data cleaning (handling NAs, filtering) and preparation (using mutate(), recode(), converting types)
  • Module 3: Descriptive statistics (frequency tables, cross-tabulations, summary stats for Likert-scale data)
  • Module 4: Inferential tests (independent two-sample t-test, paired t-test, one-way ANOVA) with examples
  • Module 5: Introduction to regression analysis and data visualization with ggplot2 (bar charts, histograms, boxplots, scatter plots)

Throughout the training, we use RStudio for hands-on practice, emphasizing reproducible scripts in the Source pane rather than only using the Console. Participants will follow along with provided dummy datasets and code examples in RStudio.

Dummy Datasets

Two dummy datasets (in both CSV and Excel formats) are provided for practice. These represent fictitious student evaluation and survey data, with Likert-scale responses and categorical demographics:

StudentEvaluations.csv

Simulated course evaluation responses from 30 students. Each row is a student's responses about a course. Columns include:

  • StudentID: Unique ID for each student (1–30)
  • Gender: Student gender ("M" or "F"; later recoded to "Male"/"Female")
  • Course: Course identifier (e.g. "CourseA", "CourseB", "CourseC")
  • Q1 – Q5: Likert-scale ratings (1–5) for five evaluation questions (1 = Strongly Disagree, 5 = Strongly Agree)
  • Overall: Overall satisfaction rating (1–5)
  • StudyHours: Numeric value of hours studied per week (e.g. 8, 10, 12 hours, etc.)
  • PreScore: Score on a pre-course knowledge quiz (percentage 0–100)
  • PostScore: Score on a post-course knowledge quiz (0–100)

StudentSurvey.xlsx

Simulated survey of 20 students about their academic experience. Columns include:

  • StudentID: Student ID
  • Year: Year of study (1, 2, 3, or 4)
  • Satisfaction: Likert-scale (1–5) rating of overall academic experience
  • Recommend: Likert-scale (1–5) rating of likelihood to recommend their program
  • StudyHours: Hours studied per week (numeric)

Both datasets contain examples of Likert-scale data (which will be treated as ordinal categorical data) and other variables for analysis. Minor missing values (NAs) are introduced in each dataset for training purposes (e.g. a couple of Q# or Satisfaction entries are left blank) to practice handling missing data.

Installing R & RStudio

Downloading R: To begin, visit the official R project website at CRAN (The Comprehensive R Archive Network). Select the download link for your operating system (Windows, macOS, or Linux). For Windows users, click "Download R for Windows" then "base" and download the latest version. For macOS users, select "Download R for macOS" and choose the appropriate .pkg file for your macOS version. Follow the standard installation prompts, accepting the default options unless you have specific requirements.

Downloading RStudio: After installing R, visit the RStudio Desktop download page. The website should automatically detect your operating system and recommend the appropriate installer. If not, scroll down to select your operating system manually. Download the free RStudio Desktop version. Remember that RStudio is an interface for R and requires R to be installed first—it won't work properly without R.

Installation tips: When installing both programs, use administrative privileges if required by your system. For Windows users, installation is straightforward with standard "Next" button clicks. Mac users will drag the application to the Applications folder after opening the downloaded file. Linux users may need to use command-line installation depending on their distribution. After installation, open RStudio (not R directly) to begin working—RStudio will automatically connect to your R installation and provide the integrated environment for your data analysis projects.

  • What is R? R is a programming language specifically designed for statistical computing and data analysis.
  • What is RStudio? RStudio is a user-friendly Integrated Development Environment (IDE) that makes working with R more efficient and intuitive.
  • Relevance to social science: R is widely used in social science research for statistical analysis and data visualization. R is free and open-source, has a large community, many packages for statistics (e.g. survey analysis, social science data), and it enables reproducible research (script-based workflow).
  • Key advantages: Researchers prefer R due to its flexibility, extensive statistical capabilities, and powerful packages for specialized analyses.
  • Important to note: R is case-sensitive (e.g., Variable and variable are treated as different objects).
  • Getting started: Commands in R are typed and executed in the Console pane, with results displayed immediately below each command.

Role of Descriptive Statistics

Why Descriptive Statistics?
  • Help us "get a feel" for the data before formal analysis
  • Understand data distribution and key characteristics
  • Essential first step in any statistical analysis
Common Descriptive Statistics in Social Science:
  • Frequencies: How many students selected each Likert option
  • Percentages: Proportion of responses in each category
  • Central Tendency: Mean, median, mode
  • Dispersion: Range, variance, standard deviation

Always start your analysis by examining descriptive statistics!

Likert Scale Considerations

Important Methodological Notes:
  • Individual Likert items are technically ordinal data
  • Strictly speaking, ordinal data should use median or mode, not mean
  • Aggregated scales (multiple items) are often treated as interval data
  • Social science convention often uses means despite statistical debate
Best Practices:
  • Create composite variables when appropriate (e.g., mean of Q1-Q5)
  • Check internal consistency (e.g., Cronbach's alpha) for multi-item scales
  • Consider both parametric and non-parametric approaches
  • Be transparent about analytical choices in reporting
Creating Composite Scores:

Introduction to Hypothesis Testing

What is Hypothesis Testing?
  • Statistical method to determine if observed differences or relationships are significant
  • Helps distinguish between real effects and random chance
  • Common in social science research to validate findings
Key Concepts:
  • Null Hypothesis (H0): No effect/difference exists (what we test against)
  • Alternative Hypothesis (H1): An effect/difference does exist
  • p-value: Probability of observing our results if H0 were true
  • Significance level: Threshold for rejecting H0 (typically 0.05)
Common Tests We'll Cover:
  • Independent t-test: Compare means of two independent groups
  • Paired t-test: Compare means of two related measurements
  • One-way ANOVA: Compare means across three or more groups

Hypothesis testing helps us make evidence-based conclusions rather than relying on observed differences alone

Independent Two-Sample t-test

Purpose:
  • Compare means between two independent groups
  • Examples: male vs. female, treatment vs. control
  • Tests if group difference is statistically significant
Hypotheses:
  • H0: No difference in means between groups (μ₁ = μ₂)
  • H1: There is a difference in means between groups (μ₁ ≠ μ₂)
Example: Comparing Overall Satisfaction by Gender

                    # Independent t-test using formula notation
                    t.test(Overall ~ Gender, data = eval_data)
                    
                    # If you want to assume equal variances (classic Student's t-test)
                    t.test(Overall ~ Gender, data = eval_data, var.equal = TRUE)
                    
                    # Alternative format with vectors
                    t.test(eval_data$Overall[eval_data$Gender == "Female"],
                           eval_data$Overall[eval_data$Gender == "Male"])
Sample Output:
Welch Two Sample t-test
            
            data:  Overall by Gender
            t = 0.57, df = 27.8, p-value = 0.57
            alternative hypothesis: true difference in means is not equal to 0
            95 percent confidence interval:
             -0.5  0.9
            sample estimates:
            mean in group Female   mean in group Male 
                          3.67                 3.53
Interpretation:
  • p-value = 0.57 > 0.05, so we fail to reject H0
  • No statistically significant difference in mean satisfaction between genders
  • The difference observed (3.67 vs 3.53) could be due to chance
  • Report: "There was no significant gender difference in satisfaction (Mfemale=3.67, Mmale=3.53, t(28)=0.57, p=0.57)"

Paired t-test

Purpose:
  • Compare means between two related measurements
  • Examples: pre-test vs. post-test, before vs. after intervention
  • Tests if the average difference is statistically significant
Hypotheses:
  • H0: No mean difference between paired measurements (μdiff = 0)
  • H1: There is a mean difference between paired measurements (μdiff ≠ 0)
Example: Testing Pre/Post Knowledge Scores

                    # Paired t-test
                    t.test(eval_data$PostScore, eval_data$PreScore, paired = TRUE)
                    
                    # Alternative: first calculate differences, then test if mean differs from 0
                    eval_data$ScoreChange <- eval_data$PostScore - eval_data$PreScore
                    t.test(eval_data$ScoreChange, mu = 0)
Sample Output:
Paired t-test
            
            data:  eval_data$PostScore and eval_data$PreScore
            t = 5.4, df = 29, p-value = 0.00001
            alternative hypothesis: true mean difference is not equal to 0
            95 percent confidence interval:
              8.0 15.0
            sample estimates:
            mean of the differences 
                             11.5
Interpretation:
  • p-value = 0.00001 < 0.05, so we reject H0
  • Students scored significantly higher after the course
  • Mean improvement: 11.5 points (95% CI: 8.0 to 15.0)
  • Report: "Students scored significantly higher after the course (Mpre=60, Mpost=71.5, t(29)=5.4, p<0.001), indicating a substantial knowledge gain."

One-Way ANOVA

Purpose:
  • Compare means across three or more groups
  • Examples: comparing courses A, B, C or different treatment conditions
  • Extension of t-test for more than two groups
Hypotheses:
  • H0: All group means are equal (μ₁ = μ₂ = μ₃ = ...)
  • H1: At least one group mean differs from the others
Example: Comparing Satisfaction Across Courses

                    # One-way ANOVA
                    result <- aov(Overall ~ Course, data = eval_data)
                    summary(result)
                    
                    # If ANOVA is significant, follow with post-hoc tests
                    # to determine which specific groups differ
                    TukeyHSD(result)
Sample Output:
            Df Sum Sq Mean Sq F value Pr(>F)
            Course       2   3.50    1.75    0.9   0.418
            Residuals   27  52.50    1.94
Interpretation:
  • p-value = 0.418 > 0.05, so we fail to reject H0
  • No statistically significant difference in mean satisfaction among the three courses
  • The observed differences in course means could be due to chance
  • Report: "Mean satisfaction did not differ significantly by course (F(2,27)=0.9, p=0.42)."
  • Since ANOVA is not significant, post-hoc tests are not necessary

Interpretation and Reporting

Effective Results Reporting:
  • Always connect statistical results to research context
  • Include test statistics, degrees of freedom, and p-values
  • Report effect sizes when possible (e.g., Cohen's d)
  • Follow discipline-specific reporting standards (e.g., APA)
Example Statements:
  • Independent t-test: "There was no significant gender difference in satisfaction (Mfemale=3.67 vs Mmale=3.53, t(28)=0.57, p=0.57)."
  • Paired t-test: "Students scored significantly higher after the course (Mpre=60, Mpost=71.5, t(29)=5.4, p<0.001), indicating a substantial knowledge gain."
  • ANOVA: "Mean satisfaction did not differ significantly by course (F(2,27)=0.9, p=0.42)."
Interpreting p-values:
  • p < 0.05: Statistically significant result (reject H0)
  • p ≥ 0.05: Not statistically significant (fail to reject H0)
  • Avoid language like "proved" or "disproved" - statistical evidence supports or fails to support hypotheses

Statistical significance ≠ practical importance. Consider effect sizes and context when interpreting results.

Other Statistical Tests

Additional Tests You May Encounter:
  • One-sample t-test: Compare a sample mean to a known value
  • Chi-square test: Analyze relationships between categorical variables
  • Non-parametric alternatives: When data violates assumptions of parametric tests
Non-parametric Alternatives:

                    # Mann-Whitney U test (alternative to independent t-test)
                    wilcox.test(Overall ~ Gender, data = eval_data)
                    
                    # Wilcoxon signed-rank test (alternative to paired t-test)
                    wilcox.test(eval_data$PostScore, eval_data$PreScore, paired = TRUE)
                    
                    # Kruskal-Wallis test (alternative to one-way ANOVA)
                    kruskal.test(Overall ~ Course, data = eval_data)
When to Use Non-parametric Tests:
  • Data is ordinal (e.g., Likert scales)
  • Data violates normality assumption
  • Small sample sizes
  • Presence of outliers that cannot be removed

Non-parametric tests are more conservative but make fewer assumptions about data distribution

Assumptions and Data Considerations

Key Assumptions for Parametric Tests:
  • Normality: Data follows normal distribution within groups
  • Homogeneity of variance: Similar variances across groups
  • Independence: Observations are independent (except paired tests)
  • Interval/ratio data: Measurements on a continuous scale
Checking Assumptions:

                    # Check normality with histogram
                    hist(eval_data$Overall[eval_data$Gender == "Female"])
                    
                    # Formal normality test (Shapiro-Wilk)
                    shapiro.test(eval_data$Overall[eval_data$Gender == "Female"])
                    
                    # Check equal variances
                    var.test(Overall ~ Gender, data = eval_data)
Likert Scale Considerations:
  • Single Likert items are ordinal, not interval data
  • Strictly speaking, non-parametric tests may be more appropriate
  • Common practice: Use parametric tests when:
    • Multiple Likert items are combined into a scale
    • Sample sizes are reasonably large
    • Data approximates normal distribution
Selecting the Right Test:
  • Independent t-test: Two separate groups (e.g., male vs. female)
  • Paired t-test: Two measurements on same subjects (e.g., pre/post)
  • ANOVA: Three or more groups to compare

Introduction to Regression

What is Regression Analysis?
  • Statistical method for modeling relationships between variables
  • Helps understand how changes in predictors relate to changes in outcomes
  • Can be used for prediction or understanding associations
  • Common in social sciences for analyzing relationships between variables
Simple Linear Regression:
  • Models relationship between one predictor (X) and one outcome (Y)
  • Finds the "line of best fit" through data points
  • Equation form: Y = β₀ + β₁X + ε
  • β₀ = intercept (Y value when X = 0)
  • β₁ = slope (change in Y for one unit change in X)
  • ε = error term (accounts for unexplained variation)
Example Scenario:

Does study time predict exam performance?

  • Outcome (Y): Post-course score (PostScore)
  • Predictor (X): Study hours per week (StudyHours)
  • Model: PostScore = β₀ + β₁ × StudyHours + ε

Fitting a Linear Model in R

Creating a Simple Linear Regression:
  • Use lm() function in R (linear model)
  • Formula notation: outcome ~ predictor
  • summary() provides detailed results
Code Example:

                    # Fit linear regression model
                    model <- lm(PostScore ~ StudyHours, data = eval_data)
                    
                    # View detailed results
                    summary(model)
Sample Output:
Coefficients:
                             Estimate Std. Error t value Pr(>|t|)    
            (Intercept)    50.00      5.00    10.0   2e-11 ***
            StudyHours      2.00      0.45     4.44  0.0001 ***
            ---
            Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1
            
            Residual standard error: 8 on 28 degrees of freedom
            Multiple R-squared: 0.413,  Adjusted R-squared: 0.392 
            F-statistic: 19.7 on 1 and 28 DF,  p-value: 0.0001
Interpreting the Results:
  • Intercept (β₀) = 50.0: Predicted score for a student who studies 0 hours
  • Slope (β₁) = 2.0: Each additional study hour predicts a 2-point increase in exam score
  • p-value for StudyHours = 0.0001: Study time is a statistically significant predictor
  • R-squared = 0.413: About 41% of variation in scores is explained by study hours
  • F-statistic p-value = 0.0001: Overall model is statistically significant

In plain language: "Each extra hour of weekly study is associated with a 2-point increase in exam score. This relationship is statistically significant and explains about 41% of the variation in scores."

Multiple Regression

Adding More Predictors:
  • Extends simple regression to include multiple predictors
  • Formula: Y = β₀ + β₁X₁ + β₂X₂ + ... + βₚXₚ + ε
  • Can control for confounding variables
  • Potentially increases explanatory power (R²)
Example Code:

                    # Multiple regression with two predictors
                    model2 <- lm(PostScore ~ StudyHours + PreScore, data = eval_data)
                    
                    # View results
                    summary(model2)
                    
                    # Compare models
                    anova(model, model2)
Interpretation Changes:
  • Each coefficient represents the effect of that variable while holding other variables constant
  • Example: "For each additional hour studied, scores increase by X points on average, controlling for pre-test performance"
  • R² typically increases with additional predictors (adjusted R² accounts for this)

Multiple regression allows for more comprehensive models but requires careful interpretation of coefficients

Introduction to ggplot2

Why Data Visualization?
  • Complements statistical analysis
  • Reveals patterns not obvious in raw numbers
  • Communicates findings effectively
  • Helps identify outliers and unusual patterns
The ggplot2 Package:
  • Powerful R package based on the "Grammar of Graphics"
  • Creates high-quality, customizable visualizations
  • Uses a layered approach to building plots
  • Widely used in data science and academic research
Basic ggplot2 Structure:

                    # Install and load ggplot2 (part of tidyverse)
                    # install.packages("ggplot2")
                    library(ggplot2)
                    
                    # Basic structure
                    ggplot(data = dataset, aes(x = variable1, y = variable2)) +
                      geom_xxx() +  # Plot type (e.g., point, bar, line)
                      labs(title = "Plot Title", x = "X-axis Label", y = "Y-axis Label")
Main Components:
  • data: The dataset to visualize
  • aes(): Aesthetic mappings (variables to visual properties)
  • geom_xxx(): Geometric objects (plot type)
  • labs(): Labels and titles
  • theme(): Visual styling and appearance

Bar Chart - Likert Scale Frequencies

Purpose:
  • Visualize frequency distribution of categorical variables
  • Perfect for Likert scale responses (1-5 ratings)
  • Shows counts or proportions for each category
Code Example:

                    # Create a bar chart of Overall ratings
                    ggplot(eval_data, aes(x = factor(Overall))) +
                      geom_bar(fill = "skyblue") +
                      labs(x = "Overall Rating", y = "Count", 
                           title = "Distribution of Overall Satisfaction")
                           
                    # Add percentages instead of counts
                    ggplot(eval_data, aes(x = factor(Overall))) +
                      geom_bar(aes(y = ..prop.., group = 1), fill = "skyblue") +
                      scale_y_continuous(labels = scales::percent) +
                      labs(x = "Overall Rating", y = "Percentage", 
                           title = "Distribution of Overall Satisfaction")
Key Features:
  • factor(Overall): Treats numeric ratings as categories
  • geom_bar(): Creates bars with height proportional to count
  • fill = "skyblue": Sets bar color
  • labs(): Adds descriptive labels

Bar charts visually highlight the most common responses and allow easy comparison of frequencies. In our example, we might see that rating "4" is most common, with few students giving the lowest rating of "1".

Histogram - Numeric Distribution

Purpose:
  • Visualize distribution of continuous numeric variables
  • Shows frequency of values within specified bins
  • Reveals shape of distribution (normal, skewed, bimodal, etc.)
Code Example:

                    # Create a histogram of study hours
                    ggplot(eval_data, aes(x = StudyHours)) +
                      geom_histogram(binwidth = 2, color = "black", fill = "orange") +
                      labs(x = "Study Hours per Week", y = "Number of Students", 
                           title = "Histogram of Study Hours")
                           
                    # Add density curve overlay
                    ggplot(eval_data, aes(x = StudyHours)) +
                      geom_histogram(aes(y = ..density..), binwidth = 2, 
                                     color = "black", fill = "orange", alpha = 0.7) +
                      geom_density(color = "red", size = 1) +
                      labs(x = "Study Hours per Week", y = "Density", 
                           title = "Distribution of Study Hours")
Key Features:
  • geom_histogram(): Creates bins and counts observations in each
  • binwidth = 2: Sets width of each bin (adjust based on data)
  • color/fill: Control appearance of bars
  • geom_density(): Adds a smoothed density curve (optional)

Histograms help identify central tendency, spread, and shape of data. Our example might show most students study 8-12 hours weekly, with a roughly normal distribution and perhaps a few outliers studying significantly more.

Boxplot - Distribution by Group

Purpose:
  • Compare distributions across different groups
  • Display key summary statistics in one visualization
  • Identify medians, quartiles, and outliers
  • Great for comparing outcomes between categorical groups
Code Example:

                    # Create boxplot of PostScore by Gender
                    ggplot(eval_data, aes(x = Gender, y = PostScore)) +
                      geom_boxplot(fill = "lightgreen") +
                      labs(x = "Gender", y = "Post-course Score", 
                           title = "Post Score by Gender")
                           
                    # Add individual data points
                    ggplot(eval_data, aes(x = Gender, y = PostScore)) +
                      geom_boxplot(fill = "lightgreen", alpha = 0.5) +
                      geom_jitter(width = 0.2, alpha = 0.6) +
                      labs(x = "Gender", y = "Post-course Score", 
                           title = "Post Score by Gender")
Understanding Boxplots:
  • Middle line: Median (50th percentile)
  • Box: Interquartile range (IQR) - 25th to 75th percentile
  • Whiskers: Extend to ±1.5 × IQR from box edges
  • Points: Potential outliers beyond whiskers

Boxplots provide an at-a-glance comparison of distributions. In our example, overlapping boxes would suggest little difference between genders (consistent with our t-test finding no significant difference). We can also easily spot differences in median, spread, and potential outliers.

Scatter Plot - Relationships Between Variables

Purpose:
  • Visualize relationship between two continuous variables
  • Show correlation and patterns
  • Identify potential outliers
  • Perfect complement to regression analysis
Code Example:

                    # Basic scatter plot with regression line
                    ggplot(eval_data, aes(x = StudyHours, y = PostScore)) +
                      geom_point(color = "blue") +
                      geom_smooth(method = "lm", se = TRUE, color = "red") +
                      labs(x = "Study Hours per Week", y = "Post-course Score", 
                           title = "Study Hours vs Post Score")
                           
                    # Enhanced version with additional features
                    ggplot(eval_data, aes(x = StudyHours, y = PostScore, color = Gender)) +
                      geom_point(size = 3, alpha = 0.7) +
                      geom_smooth(method = "lm", se = TRUE) +
                      labs(x = "Study Hours per Week", y = "Post-course Score", 
                           title = "Relationship Between Study Time and Performance",
                           subtitle = "With 95% confidence interval for regression line") +
                      theme_minimal()
Key Features:
  • geom_point(): Creates individual data points
  • geom_smooth(method = "lm"): Adds regression line
  • se = TRUE: Shows 95% confidence interval around line
  • color = Gender: Optional grouping by categorical variable

Our scatter plot would likely show a positive trend matching our regression results - students who study more tend to score higher. The slope of the red line visually represents the coefficient (approximately 2 points per study hour). The spread of points around the line relates to our R² value of 0.41 - there's a clear trend, but individual points vary from the prediction.

Customizing and Saving Plots

Enhancing Visualizations:
  • Add themes for consistent styling
  • Customize colors, fonts, and labels
  • Add annotations and reference lines
  • Adjust scales and coordinate systems
Customization Examples:

                    # Apply a theme
                    ggplot(eval_data, aes(x = StudyHours, y = PostScore)) +
                      geom_point() +
                      geom_smooth(method = "lm") +
                      theme_minimal() +
                      labs(title = "Study Hours vs Post Score")
                      
                    # Custom colors and formatting
                    ggplot(eval_data, aes(x = Course, y = Overall, fill = Course)) +

                    # Method 1: Save the last plot

                    ggsave("study_score_plot.pdf", plot = p, width = 8, height = 6)

                    ggsave("plot.jpg")  # JPEG format
